# 臉部平權網站計畫 — 個人學習總結書

**計畫期間：** 2025 年（初版）→ 2026 年（V3 完成）
**網站主題：** 陽光基金會臉部平權倡議網站（非官方、個人製作）

> 這不是一個做網站的計畫，是一個學習怎麼跟 AI 協作的計畫。網站只是副產品。

---

## 一、版本旅程

| 維度 | 初版 | V1（第一版） | V2（第二版） | V3（第三版） |
|---|---|---|---|---|
| **做了什麼** | YouTube 跟做，Perplexity 產研究內容，Claude 生圖片提示詞 | 分析 Amnesty 網站並仿效設計，加入卡片式 UI | 建立 6 工作區塊，寫了完整修正計畫書 | 三工具分工，終於把計畫帶進對話 |
| **最大問題** | 資訊散亂、對象不明、沒有導覽列 | 目標對象仍然不明確 | 計畫書做好了，但開啟 Claude Code 時沒貼進去 | — |
| **當時以為** | 有內容就夠了 | 有設計就夠了 | 有計畫就夠了 | — |
| **真正學到** | 結構先於內容 | 參考物件要明確、要說你喜歡哪幾點 | 計畫要帶進對話才算用了 | Context 是對話的燃料 |

**一句結論：每一版的失敗，都是少做了「下一版才補上」的那件事。**

---

## 二、我真正搞懂的那些瞬間

每個「啊哈時刻」格式：【情境】→【以前以為】→【後來發現】→【現在怎麼做】

---

### 啊哈 1：計畫書要帶進對話，不是放著

**【情境】** V2 花了很多時間做出完整修正計畫書（臉部平權網站改版完整計畫書 v2），裡面有診斷、刪除清單、修正清單、新增清單、優先順序。但打開 Claude Code 開始改的時候，沒有把計畫書貼進去。

**【以前以為】** 做好計畫就算準備好了。

**【後來發現】** Claude 每次開新對話都是全新開始，你上一段對話說的所有事它完全不知道。計畫書放在 docs 資料夾裡，對 AI 來說等於不存在。

**【現在怎麼做】** 每次開新對話，第一件事就是把計畫書的內容複製貼進去作為開場 context，然後再說「請按照這份計畫書執行第 X 項」。

---

### 啊哈 2：AI 不是一個，是一個分工團隊

**【情境】** 前三版都是一個 AI 問到底。V3 才開始用不同工具做不同事。

**【以前以為】** 一個 AI 能做所有事，為什麼要換來換去？

**【後來發現】** 每個工具擅長的事不一樣：Perplexity 會搜尋最新資料、Claude Chat 適合長對話和計畫書討論、Claude Code 適合真正動程式碼。強迫一個工具做三件事，效率很低，也很容易出錯。

**【現在怎麼做】** V3 的三步驟工作流：
1. Perplexity → 找問題、搜尋資料
2. Claude Chat → 討論計畫、定位問題在哪
3. Claude Code → 執行修改

---

### 啊哈 3：網站是切出來的，不是寫出來的

**【情境】** V2 把整個網站切成六個工作區塊（01-hero-and-nav、02-intro-and-principles、03-six-cards-and-overlay、04-stories-video-links、05-footer、06-design-system）。

**【以前以為】** 網站是一個整體，AI 就照著做就好。

**【後來發現】** 如果你說「幫我改網站」，AI 可能動到你不想動的部分。如果你說「只改第 4 區，不動其他部分」，AI 的穩定度就高很多。

**【現在怎麼做】** 改東西之前，先在心裡畫出邊界：這次只改哪一塊，哪些絕對不能動。說清楚，AI 才不會亂跑。

---

### 啊哈 4：「清楚」不是你覺得清楚，是 AI 不會猜錯

**【情境】** V3 建立「AI 執行適配版」計畫書，裡面第一頁就是規則清單，連結格式也統一用 `{{SUNSHINE_OFFICIAL_URL}}` 這種 placeholder。

**【以前以為】** 說「幫我加連結」就夠清楚了。

**【後來發現】** 你省略的每一個細節，AI 都會自己腦補一個答案。你說「加連結」，它可能加一個根本不存在的 URL。你說「待填連結用 `{{SUNSHINE_OFFICIAL_URL}}`」，它就會留著等你填。

**【現在怎麼做】** 把你不想讓 AI 決定的事，全部明確寫出來當規則，放在每次對話的開頭。

---

## 三、怎麼思考網站

三個思維模型，不是技術知識，是看網站的眼睛。

---

### 模型 1：網站是給「某一種人」在「某一種狀態」下看的

不是「所有人」，是「某種人」。

這個計畫花最久時間才搞清楚的問題，就是「目標對象」。V1 設計做完了，對象還是不清楚。一直到 V2 才明確定出：完全沒聽過這個議題的一般人，手機使用佔 2/3。

在你明確這個之前，所有設計決策都是猜的。

問自己的問題：「我的網站是給『第一次聽到這個議題』的人看，還是給『已經關心但不知道怎麼行動』的人看？」這兩種人需要完全不同的第一句話。

---

### 模型 2：資訊有層次，不是清單

網站不是把所有重要的事列出來。而是設計一條「注意力路徑」：

```
第一眼看到什麼 → 產生什麼問題 → 往下滑找答案 → 找到後想做什麼
```

初版的問題就是：清單很完整，但路徑不存在。讀者進來不知道從哪裡開始，也不知道為什麼要繼續往下看。

---

### 模型 3：每個區塊只做一件事

V2 之後，每個工作區塊都有自己的單一任務。

問自己：「這個 section 存在的唯一理由是什麼？」如果答案超過一個，就是還沒切乾淨。

例如：Hero 區的唯一任務是「讓第一次進來的人在 10 秒內懂這個網站是什麼」，不是同時介紹所有主題、也不是推銷捐款。

---

## 四、怎麼跟 AI 溝通

這是整份文件最核心的部分。七個方法，每個都附上當時我實際用的提示詞。

---

### 方法 1：先問「是什麼、社會怎麼看」，再問「給我幾個主題」

分兩步走，讓 AI 有脈絡再輸出有用的東西。

**我說了什麼（初版，問 Perplexity 第一步）：**
```
哈囉！我想知道現在台灣對於臉部平權的的問題：
1. 臉部平權是什麼
2. 社會上有哪些誤解
我想知道大眾是怎麼想的。幫我搜尋一下吧
```

**然後第二步：**
```
嗯那現在我也想為臉部平權出一份力，那你可以向我介紹一些大眾最想了解、
最不熟悉，或是基金會宣傳的方向，並且幫我為這個標題取個名，
推薦你標題統一成「問題式」或「知識式」命名，
此外我需要六到八個這樣的主題，並且加上一個平權日有關的主題。
以下僅供參考不用抄...
```

**為什麼有效：** 第一步讓 AI 先建立脈絡，第二步才叫它輸出清單。順序反過來，清單會很空洞。

---

### 方法 2：給 AI 一個你看過的「參考物件」，說你喜歡哪幾點

不要說「幫我設計一個專業的網站」，因為 AI 不知道你心裡的「專業」長什麼樣。

**我說了什麼（V1 開場，把 Amnesty 網站分析整合成一個提問）：**
```
你好，我現在要下一個重大的決定，我現在想重新地去設計我明天宣傳要使用的網站，
首先我學到了如果是在路邊的話，還是簡單易懂的宣傳，別人比較看得下去...
我有一個參考的網站連結：https://www.amnesty.tw/issues，他的優點有以下幾種
我們在設計網站時需要特別注意的：
1. 他的設計讓我在 10 秒內就能明白這個連結在介紹什麼
2. 他能在主畫面中設計一個按鈕來開啟一個選單
3. 他的設計是讓我第一眼放鬆，並能一眼就知道這裡的主題
4. 卡片式的設計能一眼看出大綱與內容
5. 可以設計圖標式的連結
所以現在我需要你盡可能地吸收這些優勢與我看中的點，
和我一起來設計這個網站，即使這是不小的挑戰，也讓我們一起加油吧！
```

**為什麼有效：** 給 AI 一個真實存在的參考點，再說你喜歡哪幾個特點。AI 有了具體目標，不需要猜你的喜好。

---

### 方法 3：把多個問題列出來，說「這次只改這些，不動其他的」

**我說了什麼（V1 修改過程中）：**
```
恩我感受到了這次的巨大進步與可行性，我想這個目標是正確也能夠被執行的，
那好我們繼續加油，首先我觀察到了幾個點：
1. 請你試著點擊看看每一個連結與圖片，因為我目前的使用中，
   很多的連結和卡片等都是無法順利進入的，連結常常會顯示錯誤
   「您的連線並非私人連線」，我想這是個很嚴重的問題，
   所以我們這次的目標就是盡可能的確保這些連結的可使用性
2. 因為這次只是單純的宣導，不需要大家有多麼遠大的理念，
   幫我將網站最後的捐款給移除吧
這次先這樣試試，謝謝有你，讓我們開始吧
```

**為什麼有效：** 把「問題清單 + 這次只改什麼 + 不改什麼」放在一起說。AI 的執行範圍就被限制住了，不會去動你沒叫它動的地方。

---

### 方法 4：問「AI 無法生成、需要我來補的部分是什麼」

這是 V2 和 V3 都用到的一個關鍵句。

**我說了什麼：**
```
幫我找出 AI 無法生成的部分，也就是我提供給它會更好的部分
```

**為什麼有效：** 這句話讓 AI 主動告訴你它在哪裡缺少真實資料（例如：正確的官網連結、有來源的數據、真實的案例故事）。你就知道接下來要補什麼，不用從頭猜。不問的話，AI 會自己腦補一個聽起來合理但可能是假的答案。

---

### 方法 5：補充資料時，把格式和文字都給好，不留 AI 詮釋空間

**我說了什麼（V3 補充核心資料時節選）：**
```
1. 改成：在台灣，每 5 人中就有 1 人因外貌遭受不友善對待
2. 避諱詞清單：哪些詞會讓人很不舒服？
3. 這個網站來自於沒有合作的大學學生，應未受到服務學習臉部平權宣導的啟發，
   而決定出一份力在社群上用以向大家介紹宣傳的網站（非官方個人整理），
   無指導老師全由 AI 生成和學生構想...
```

**為什麼有效：** 不只是說「補這個」，而是連格式、具體語句都直接給出來。AI 收到的是「把這個放進去」，不是「幫我想一個說法」。這樣輸出才會是你要的樣子。

---

### 方法 6：研究多個主題時，告訴 AI「每個主題不要混進其他主題的內容」

**我說了什麼（V3 用 Perplexity 深入研究八大主題時）：**
```
我想為台灣臉部平權發聲，所以我搜尋了以下的八個主題，
讓我們一項一項來深入研究其中的問題和社會的現象議題，
但請注意在生成其中一項的資料時，最好不要過多次的涵蓋到其他的主題，
不然我也不知道其他的主題要探討什麼，
所以我也找出了一些建議的核心概念，你不需要照抄，參考就行，
最好你可以再深入其他相關的內容...
```

後面附上八個主題清單，以及每個主題的寫作規則：先講人的處境而非抽象議題、加入情緒重量、每段只留一個核心畫面。

**為什麼有效：** 解決了一個常見問題：AI 為了讓每個主題看起來完整，會把所有相關內容都塞進去，結果八個主題都長得差不多。明確說「不要混」，各主題才會有各自清楚的焦點。

---

### 方法 7：當 context 已經夠了，最後只需要一句話

**我說了什麼（V3 最後推進到 v3.1）：**
```
好幫我產出 v3.1
```

**為什麼有效：** 這不是懶，是前面把功課做足了。當所有規則、修正說明、資料補充都已經在這次對話裡了，最後只需要叫 AI 執行。重點永遠是前面的 context，不是最後那句話。

## 五、內容寫作原則

從這個計畫的 Perplexity 研究和 Claude 生成過程中提煉出的四條原則。

---

**1. 先講人的處境，不是議題本身**

讓讀者第一眼看到的是「人」，不是「制度」或「議題標籤」。

- 錯誤開頭：「臉部平權是一個關於外貌歧視的社會議題...」
- 正確開頭：「你有沒有因為一張照片，決定要不要投這份履歷？」

---

**2. 每個 section 只有一個核心畫面**

不是說只有一張圖，是說讀者看完這個區塊之後，腦袋裡只留下一個印象。
如果看完之後有兩個印象，就是這個區塊還沒切乾淨。

---

**3. 製造資訊缺口，讓人想繼續讀**

好的標題不是總結答案，是製造問題。讓讀者有「這是什麼意思？」的感覺，然後在下面找答案。

答案已經在標題裡的文章，讀者不需要繼續看了。

---

**4. 句子要短、要硬、要清楚**

- 不好的：「請試著考慮在文字中加入一些能夠引起讀者情感共鳴的表達方式。」
- 好的：「加入情緒重量。」

這個原則也適用在跟 AI 的提問上。你的問題越清楚，它的回答越精準。

## 六、下次再做類似計畫，我會先做什麼

```
□ 先定義「這個網站是給誰、在什麼狀態下看」，再開始任何設計

□ 把網站切成工作區塊，給每個區塊一個任務描述
  （例如：Hero 的唯一任務是讓陌生人在 10 秒內懂這是什麼）

□ 建立一份「AI 對話開場規則」：
  - 每次新對話第一件事就把計畫書或規則貼進去
  - 格式越明確越好，例如用 {{PLACEHOLDER}} 代替待填內容

□ 用三工具分工：
  - Perplexity：找問題、找資料
  - Claude Chat：討論計畫、生修正計畫書
  - Claude Code：執行修改

□ 修改前先寫修正計畫書，修改時把計畫書帶進對話

□ 每次改完，問：「幫我找出 AI 無法生成的部分，需要我補的地方」
  → 用真實資料補進去，再跑下一輪
```
