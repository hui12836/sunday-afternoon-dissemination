## 4. Accordion / 展開問題功能修正

### 問題
「展開問題」區塊疑似有樣式存在，但互動可能失效。

### 可能原因
- JS selector 改名後沒同步更新
- `data-target` 對不到內容區塊 ID
- button 被改成非互動元素
- 展開內容被 `display: none` 後沒有成功切換

### 修改建議
使用一致的 accordion 結構，不要混用不同命名方式。

### 建議 HTML 結構
```html
<button class="faq-toggle" aria-expanded="false" aria-controls="faq-1">
  展開問題
</button>
<div id="faq-1" hidden>
  ...
</div>
```

### 建議 JS 邏輯
```js
document.querySelectorAll('.faq-toggle').forEach((btn) => {
  btn.addEventListener('click', () => {
    const id = btn.getAttribute('aria-controls');
    const panel = document.getElementById(id);
    const expanded = btn.getAttribute('aria-expanded') === 'true';

    btn.setAttribute('aria-expanded', String(!expanded));
    panel.hidden = expanded;
  });
});
```

## 目前最像程式碼錯誤的問題

### 圖片容器高度或裁切設定錯誤

截圖中的情境卡出現大片空白，但 HTML 裡又確實有圖片路徑，最常見原因是：

- `img` 有載入，但被 `object-fit` / `object-position` 裁得太過頭。
- 卡片圖片區高度固定過高，圖片本體太小。

# 網站比對檢查結果（手機截圖 vs 線上網站）

## 1. 情境卡圖片顯示異常

### 問題
手機截圖中，部分情境卡只看到底部裝飾與大片留白，沒有正常呈現完整情境圖片。

### 判斷
這不像文案問題，比較像圖片容器、裁切方式或路徑設定錯誤。

### 可能原因
- `img` 有載入但被 `object-fit: cover` / `object-position` 裁切錯誤
- 圖片容器高度固定太高
- 圖片實際尺寸與容器比例不合
- 圖片層被背景色或其他區塊蓋住

### 修改建議
- 檢查情境卡圖片容器高度
- 檢查手機 breakpoint 下的 `object-fit` 和 `object-position`
- 確認圖片不是同時用 background-image 與 `<img>` 混用
- 實際在手機 DevTools 查看圖片是否成功載入

### 建議 CSS
```css
.story-card img {
  width: 100%;
  height: 220px;
  object-fit: cover;
  object-position: center;
  display: block;
}
```

# 網站比對檢查結果（手機截圖 vs 線上網站）

## 1. 情境卡圖片顯示異常

### 問題
手機截圖中，部分情境卡只看到底部裝飾與大片留白，沒有正常呈現完整情境圖片。

### 判斷
這不像文案問題，比較像圖片容器、裁切方式或路徑設定錯誤。

### 可能原因
- `img` 有載入但被 `object-fit: cover` / `object-position` 裁切錯誤
- 圖片容器高度固定太高
- 圖片實際尺寸與容器比例不合
- 圖片層被背景色或其他區塊蓋住

### 修改建議
- 檢查情境卡圖片容器高度
- 檢查手機 breakpoint 下的 `object-fit` 和 `object-position`
- 確認圖片不是同時用 background-image 與 `<img>` 混用
- 實際在手機 DevTools 查看圖片是否成功載入

### 建議 CSS
```css
.story-card img {
  width: 100%;
  height: 220px;
  object-fit: cover;
  object-position: center;
  display: block;
}
```
```

```md
## 2. 頁內跳轉可能被固定 header 擋住

### 問題
部分截圖看起來像從段落半中間開始，不像自然閱讀位置。

### 判斷
可能是頁內錨點跳轉後，被固定頂部導覽列遮住標題。

### 修改建議
為所有目標區塊增加 `scroll-margin-top`。

### 建議 CSS
```css
section[id] {
  scroll-margin-top: 88px;
}
```

```md
## 3. 線上版本與你預期內容可能不同步

### 問題
手機截圖中可見某些標題，但線上 HTML 抓取時找不到完全相同文字。

### 判斷
這表示截圖內容與目前部署版本可能不是完全同一版。

### 可能原因
- Netlify 還在快取舊版
- 手機瀏覽器快取未更新
- 本地端修改後未完整部署
- 某些文案來自 JS 動態插入，導致靜態 HTML 與畫面不一致

### 修改建議
- 強制重新部署一次
- 清除手機瀏覽器快取
- 用無痕模式重開網站
- 比對 Netlify 最新 deploy 與本地檔案差異

```md
## 4. 網站內仍殘留模板字串

### 問題
HTML 中仍可搜尋到 `{{` 字串，表示仍有模板殘值未清乾淨。

### 影響
- 可能造成部分文字、連結或圖片變成錯誤值
- 未來還會出現新的顯示異常

### 修改建議
在正式部署前，全面搜尋：
- `{{`
- `}}`
- `href=""`
- `src=""`
- `onclick=`

### 建議做法
部署前加一個檢查腳本，若出現模板字串就阻止上線。
