## 網站初衷(必讀前言)

本網站由淡江大學學生編製。

在實際生活中，我們可覺到一般人對這個議題的冷漠和疲乏，不過我以為這是正常的。因此我們檢視了自己的態度與語調，在之後的宣傳中特別注意這些方面，預助我們完了現場宣傳活動。

但活動結束後我們意識到：**現場那麼多人，真正被我們宣傳到的，也只是其中一小部分。**

如果想要讓新面孔能夠了解這個概念，最快的方式就是製作一個網站──讓沒空的人能夠了兩就走，在社群上傳播，在空間裡能夠慢慢閱讀。**這就是我們設立這個網站的初衷。**

> **編者補充:** 這段初衷建議放在頁尾「關於本站」後，而不是首頁 Hero。因為首頁要先解決議題，讓讀者先自己有共鳴；等他認同之後，看到「這是一群學生自發做的」更有感，而不是一開始就用學生身份壓過議題本身的份量。

## 一、整體診斷與第三輪回饋重點

### 核心判斷（三輪回饋後的展開結論）

網站目前最大的問題**不是不夠亮，而是首頁任務不夠定一、資訊優先順序不清楚、內容重複不夠節制**。

v1 找出「動線問題」，v2 把內容結構整理好，但仍保留了太多區塊；**v3 的核心任務是「削到只剩主線」**──首頁唯一主任務是：讓第一次來觸者在第一次得到第一次理解。

### 現有優點（該保留的部分）

- **色彩辨識度高**：黑底橙點讓議題感與活動感更明確
- **卡片化版面**：對手機閱讀比長段文字友好
- **內容層次清楚**：已分成認識概念、故事案例、「要/不要」延伸敘事等模組
- **主標語記憶點**：「不要看一張臉，要看一個人」扣住議題都

### 2-3 陽光基金會服務項目（次級頁使用，首頁保一句）

**首頁使用版本（只看一句）：**

> 陽光基金會長期服務的當事及家屬，並經由倡議、教育與支持服務推動臉部平權。

**次級頁完整文案（各段）：**

#### 服務 1：全方位重建

陽光基金會是台灣專門服務的顏面傷及顏面傷者的公益組織，整合社工、復建、心理、社會等專業領域，協助當事人改善的生活功能、服務及就業能力，重新融入社會的生活。

服務核心不是只有治療，而是「全方位重建」：包含的復健重建、心理輔導、經費補助、居家復建護理，以及就學與就業輔導，讓顏面傷者在身體、心理與生活功能上都有支持。

> 「服務項目包含的復健重建、心理輔導、經費補助、居家復建護理、就學和就業輔導等。」
> —— 陽光基金會服務介紹

#### 服務 2：顏傷者及外觀不同者的負責機構

從 1981 年成立起，陽光基金會以以燒傷者及顏面傷者為主要服務對象，包含先天性（如唇顎裂、招耳缺、黑色素磁、顏面血管瘤）與後天性可見及（如疤痕傷、撕裂顏面外傷）引起的外觀差異。

對外觀不同的成人、兒童及其家長，都有提供心理諮詢、同儕支持、手術資訊、就學支持和就業輔導，協助他們面對長期性、復健與社會適應之中的壓力。

#### 服務 3：就學與就業支持

在就學階段，陽光派遣專業社工與諮詢師進入校方，協助師生與家長正確認識顏面傷者的傷，疏減遭遇度和心態歧視壓力，幫助孩子在心理調適、人際互動與學校的活中更有信心。

在就業階段，基金會提供就業諮詢、工作評估與資源連結，協助傷者及服務與就業準備與能力，並進行就業前訓練及雇主倡議服務，讓他們有機會穩定就業、重新立足職場。

#### 服務 4：倡議總結入

多年來陽光基金會一方面協助顏傷者及家屬獲得的、心理與建設服務，一方面透過校园宣導、政府與企業倡議、社會行路等方式推動「臉部平權」，希望外觀差異不再成為一不公平對待的起因。

## 三、組織決策結論

### 3-1 用詞決策

- **統一使用「顏面傷者」** — 不使用「臉部差異者」或其他替代詞
- **網站主流述不使用「患者」、「病友」** — 除非在服務與聯絡中（例如醫療、復建）

### 3-2 避諱詞清單

#### ✗ 絕對禁止（網站上完全不用，即便是引用）

- 毀容
- 相貌
- 醜陋、醜陋物
- 殘缺醜、殘障人、後天性
- 龍頭獸、鬼獸、妖獸等岐視性詞彙
- 「看了讓你恐到」、「放到小孩」等把當事人當作恐嚇物件的說法

**為什麼:** 這些詞不是中立描述，而是直接把人當成「不完整」、「可憐」、「不該出現在公共空間的存在」。

#### ⚠️ 謹慎使用（只在「解釋/修正」語境下出現）

| 謹慎詞 | 建議替代 |
|-------|---------|
| 殘、殘疾 | 「被覺得不好的」等直接說法 |
| 缺陷、殘缺 | 「外觀有差異」、「身體條件不同」 |
| 正常人、一般人 | 「多數人」（避免暗示顏面傷者是「不正常」） |

**示範對照：**
> ✗ 不可寫：「他因為長相很醜，被說像妖怪」。
> ✓ 改可寫：「他曾經被用後來讓輕辱」或「有人曾說或像妖怪的方式嘲笑」。

- **主線：** 能動 + 行動指引
- **輔以：** 到同感性標語和收尾帶感線條

### 文字色使用規則

| 用途 | 變數 | 備注 |
|------|------|------|
| H1/H2 大標 | `var(--text)` | 不得深灰棕 |
| 一般段落內文 | `var(--text-muted)` 或 `var(--text)` | 重要段落用深色 |
| 小標籤 / label | `var(--primary-hover)` 或 `var(--secondary)` | 每次不用，不要每個區塊混用 |
| 可點連結 | `var(--primary-hover)` | hover `#8F5310` |
| 大數字 / 關鍵詞 | `var(--primary)` | 重點強調 |
| 深色底上主字 | `var(--text-inverse)` | 頁尾等深底版 |
| 深色底次字 | `#D8CEC2` | - |

### 使用四大規則

**規則 1：橙色只拿來標「重要」**
橙色保留給：CTA、數字、區塊小標、關鍵詞、連結、不要讓整段文字都橙色。

**規則 2：正文不要用淺灰**
米白風容易誤認是文字太淡。正文完全就是 `var(--text)` 或 `var(--text-muted)`，不要更淡。

**規則 3：綠色是輔助，不是第二主角**
綠色的工作是讓橙色不顯得一個人全部，不是要和橙色競爭。

**規則 4：同一個區塊最多兩個重點色**
一個區塊通常就是：主文字深棕 + 一個主強調色，最多加一個淺底 highlight。

### 執行時的過渡建議

若改版是「新版本漸進替換」，請一次把 CSS 變數全部換完，不要新舊並存（避免米白區塊與黑底區塊視覺衝突）。
