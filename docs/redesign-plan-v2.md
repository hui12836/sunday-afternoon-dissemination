## 零、專案基本設定（本次新增）

這一章是所有設計決策和框架決策的「底層」，後面所有建議都建立在這些設定之上。

### 0.1 網站身份

| 項目 | 設定 |
|---|---|
| 網站性質 | **非官方個人整理頁** |
| 製作者身份 | 大學學生，無任何師徒關係 |
| 產出方式 | AI 生成 + 學生審視 |
| 商業屬性 | 不公開 |
| 著作性質 | 未正式徵得陽光基金會同意，但引用範圍限定於官方公開資料 |
| 營業性質 | **完全非營利**，頁面不含廣告、不含任何營利行為 |

**核心引用原則**：上述頁面屬於公共性資訊整理。「不涉及營利」，資料來源來自基金會公開發布的內容，理論上不屬於可被引用的著作使用範圍。但 footer 必須明確聲明本頁的「整理用途」，不可讓人誤解為是官方頁面。

### 0.2 目標讀者

| 層級 | 對象 | 觸達方式 |
|---|---|---|
| **主軸讀者** | 完全沒概念的一般人，偶好奇對這個主題。平常不會主動找到 | 用他們聽得懂的日常語言寫在頁面主文框架 |
| **副軸讀者** | 教師 / HR / 醫療人員等特定職業角色 | 收進頁面最後的「延伸了解」入口，不混入主文框架 |

**核心目標**：讓主軸讀者在 **5～10 分鐘內**明白三件事：
1. 什麼是臉部平權
2. 這個議題為什麼跟每個人都有關
3. 我在現實中可以做什麼改變

### 0.4 核心訊息（整頁的靈魂句）

> **「不要看一張臉，要看一個人」。**

這句話的設計原因：

- **簡短好記**：離開頁面後他能記住
- **視角正確**：講的是「你對待人的方式」而不是「你該怎麼幫他們」，避免施捨感
- **可重複使用**：能放進 Hero、卡片、「你不不要」框架裡面、頁腳訴求行動

**建議使用位置**：

| 位置 | 建議用法 |
|---|---|
| Hero 副標或背景 | 作為主要視覺引導 |
| 你不不要框 | 作為框架標語 |
| 頁尾 | 「離開這頁之後，希望你記得：不要看一張臉，而是看一個人」。 |

## 一、核心診斷摘要

### 1.1 關於這份網站現在的狀況

網站**不是不好的**，而是處於「內容已具雛形，但資訊架構、信任感與互動邏輯還不夠成熟」的階段。

對第一次進來的新客來說，網站沒有在 5 秒內回答三個關鍵問題：

| 關鍵問題 | 目前狀況 |
|---|---|
| 這是什麼網站？ | ✓ 大致對，「臉部平權議題頁」 |
| 為什麼我現在要看？ | ⚠️ 不夠強 |
| 我可以從哪裡開始？ | ⚠️ 不夠明確 |

### 1.2 五個核心問題

| # | 問題 | 具體現象 |
|---|---|---|
| 1 | 首屏訊息太滿，行動不夠明確 | 有定位，但沒有一個強烈的主 CTA |
| 2 | 多張卡片內容位缺少層次 | 卡片並列「資訊」，缺少「學習路徑」 |
| 3 | 文框過長，脈絡與資訊混在一起 | 同段文字夾雜定義、說明、引言、說明各種功能 |
| 4 | 信任設計不足 | 引用數據但缺來源、年份、原始連結 |
| 5 | 頁面節奏太一致 | 缺少「快速瀏覽／深入閱讀／立刻行動」三種節奏的劃分 |

### 1.3 整體觀察

這類**知識卡議題型網站**的一個容易被忽視的特性：使用者通常不會從頭讀到尾，而是「跳著看」或「有趣往下」。所以重點不是把內容寫得更豐富，而是讓**每一頁都能單獨被看懂、單獨給出下一步**。目前內容飽度其實已經足夠，真正該做的是「把我已有的東西重新分層」，而不是加加內容。

## 二、全頁改版大四原則

在進入每個區塊建議前，先記住這四條原則，後面每一個都會對應到這個點：

1. **一頁一問題** — 每個區塊只回答一個核心問題，不要在同一頁同時做到：定義 + 卡議 + 教學 + 行動。
2. **一頁一出口** — 每個區塊讓者要知道下一步去哪，不能讓者就這樣淡
3. **從淺到深展開** — 不要一開始就把所有完整章節擠進，有的該分層遞進
4. **每條數據、事例、建議都要有來源** — 這是建立公信力的關鍵

## 三、用語深度分級（本次新增）針對目標讀者：

因為主軸讀者是「完全沒概念的一般人」，以下是全頁框架遵守的語言深度原則。

### 3.1 概念語言（改成「庶民」版本）

| ✗ 避免（太學術） | ✓ 建議（庶民） |
|---|---|
| 臉部平權：是一套多元包容差異對待任何人。 | 臉部平權：是一個「因為外觀長得不一樣，就覺得那個人比較差、比較不值得尊重」。 |
| 每五人就有一人，外觀歧視發生的 20%。 | 每五個人中，就有一個人曾經歷至少一項因外觀產生的不公平對待。 |
| 企業面試流程中，外觀篩選不符合平等就業精神。 | 你可能曾在面試時看到一個外觀不一樣的人，但你應該把注意力放在他的能力。 |

### 3.2 案例深度（聚焦日常場景）

| ✓ 要放的場景 | ✗ 不該放的場景 |
|---|---|
| 日常簡單、求職、校園、社區觀感 | HR 的法律義務定義、醫療復健保險給付、職業裝扮計算 |

### 3.3 行為框架（每個人都能做）

| ✓ 要放的行為 | ✗ 不該放的行為 |
|---|---|
| 不評論、不追問、不訕笑、不瞄視<br>要以平常心、要自然互動、要把眼光放對 | HR 應該改制度、法律條文、制度設計 |

### 3.4 副軸讀者處理方式

在頁面最後「延伸了解」加入一組針對特定角色的延伸閱讀，**給特定讀者的延伸閱讀**

- 給教師看的行動指引
- 給 HR 的職場建議
- 給醫療人員的相關說明

用時：連結形式展現，**不直接寫進主文框架**。

### 區塊 5 — 「不不不要」五組框（全頁最大亮點）

> **⚠️ 優先級最高**：使用者已確認「不不不要是最大亮點主題」，這一組的視覺感與質感應高於其它組。

| 項目 | 內容 |
|---|---|
| **區塊目的** | 把抽象議題轉換成最容易執行的行動框架 |
| **目前問題** | 內容清楚但視覺組、文章、表，不能快速吸收的行動清單 |
| **修改方向** | 改成左右對稱欄，**每條必帶「為什麼」** |

**區塊標題建議**：`第一次接觸到這類情況，可以怎麼互動？`
**區塊說明建議**：`自然，不等於淡漠；尊重，也不等於要迴避。多數情況，平常心就是最好的開始。`

#### 建議欄（翻轉對稱）

| ✗ 不要 | ✓ 不不 |
|---|---|
| **要以平常心對待**<br>把對方當做一般人相看，不需要過度小心翼翼。 | **不評論**<br>長時間注視會讓人感到被放大檢視。 |
| **要自然地互動**<br>正常對話、正常發出接觸，就是最基本的尊重。 | **不追問隱私**<br>除非對方主動講起，否則不要直接追究原因或經過。 |
| **要把眼光放對自己趣**<br>把注意力和眼光放在這個人是誰，而不是外觀差異。 | **不瞄視**<br>竊笑、發出示意性私下評論，都可能造成傷害。 |

#### 每條固定格式（三行制）

```
第一行：標題（動詞開頭）
第二行：說明（做什麼 / 不做什麼）
第三行：為什麼 — 本次確認必加
```

**範例**

```
不評論
長時間注視會讓人感到被放大檢視。
為什麼：因為長時間注視會讓人覺得自己被標記、被檢查。
```

```
不追問隱私
除非對方主動講起，否則不要直接追究原因或經過。
為什麼：這是隱私，而不是你好奇的對象。
```

#### 核心訊息建議放這裡

可以在「不不不要」欄的最後放一個引導框：

> **不要看一張臉，要看一個人。**

**CTA 建議**：`看完整行動建議`
**分享設計**：區塊可標記可分享的小型卡，增加傳播性
